## 핵심 아키텍처 (현재 확정 — 항상 최신 유지할 것)

### 데이터 흐름
```
Python 엔진(GitHub Actions, cron-job.org가 트리거) → data.json → GitHub Pages(index.html)
```

### 시세 데이터 소스 (2026-06-22 변경)
- **국내 지수(코스피/코스닥) + 국내종목 8개**: **네이버금융** (`polling.finance.naver.com`, `api.finance.naver.com/siseJson.naver`) — 로그인 불필요, 한국거래소 데이터 직접 미러링, 실측 검증 정확함 확인. 실패시에만 야후 폴백.
- **미국 지수/주요주/환율/원자재/채권**: 야후파이낸스 그대로 유지 (검증 문제없었음)
- **이유**: 야후의 한국 종목 데이터(`meta.chartPreviousClose`)가 부정확해서 등락률이 실제 대비 5배까지 틀어지는 버그 발견(SK하이닉스 +26.53% vs 실제 +5.72%). `engines/market.py` 참고.
- **⚠️ 미해결 후속**: 브라우저에서 직접 야후를 부르는 부분(`index.html`의 `yfetch` — ETF카드/종목상세차트/카드 기간전환버튼)은 아직 야후 그대로라 같은 버그에 노출될 수 있음. CORS 때문에 브라우저에서 네이버를 직접 못 불러서 보류 중.
- pykrx(한국거래소 공식, 투자자별 순매수용)는 KRX_ID/KRX_PW 로그인 정보가 없어서 **여전히 미작동** (코드는 있지만 항상 실패).

### 유튜브 (2026-06-22 재설계)
- 채널당 최신 1개만 보던 구조 → **오늘(00:00~23:59 KST) 올라온 전체 영상**을 수집·표시하는 구조로 변경. 자정 지나면 자동 리셋.
- `data.json`의 `youtube` 배열은 채널별 dict가 아니라 **영상별 리스트**(채널당 여러 개 가능), `videoId` 기준으로 dedup.
- 신규 영상은 `isNew:true` 플래그 → 웹에서 빨간 New 뱃지, 텔레그램 발송 대상.
- **채널별 `summaryMode`로 비용 관리** (`engines/youtube.py`의 `CHANNELS` 설정):
  - `'full'`: 자막(Supadata) 조회 + 전체 분석 요약 (개인채널 3곳 — 영상 수 적음)
  - `'title'`: 제목만 AI요약, 자막조회 안 함 (미디어채널 3곳 — 하루 10~100개 이상 올라와서 비용 부담)
  - `fullKeywords`: 제목에 키워드가 있으면 `'title'` 채널도 그 영상만 `'full'`로 승격 (예: 한국경제TV의 "당신이 잠든사이" 코너)
  - **규칙**: 새 채널을 추가하거나 기존 채널 설정을 바꿀 때는 "하루에 몇 개 올라오는지 + summaryMode를 뭘로 할지"를 달봉즈님께 먼저 물어볼 것
- **⚠️ 비용 모니터링 필요**: 위 변경으로 Anthropic API 호출이 기존 대비 크게 늘어남(미디어채널 기준 채널당 1개/일 → 10~100개/일). 며칠 지켜보고 추가 조정 여부 결정.

### 스케줄 (2026-06-22 확정)
- **cron-job.org가 유일한 스케줄 소스** (`invest-morning/yt1/yt2/yt3/email/full` 6개 잡, GitHub API `workflow_dispatch` 호출)
- `update.yml`에는 **GitHub 네이티브 `schedule:` 없음** — 예전에 cron-job.org 도입 후에도 같이 남아있어 중복실행 버그가 났던 적 있어서 완전히 제거함. 절대 다시 추가하지 말 것.
- 시각(KST): yt1 07:05 / yt2 08:08 / **morning 08:30**(2026-06-22 22분→30분 변경) / yt3 10:12 / email 15:07 / full 18:14
- `run.py`의 `get_mode()`가 cron-job.org의 `SLOT` 파라미터로 모드 결정. `GITHUB_SCHEDULE` 폴백 로직은 죽은 코드지만 무해하니 방치.

### 포트폴리오 (2026-06-21 아키텍처 확정, 2026-06-22 코드 분리)
- **localStorage가 항상 단일 소스** (구글시트 연동 여부와 무관하게 항상 동일하게 동작). 구글시트 자동동기화는 포기함(워크북 셀 한도 문제, 아래 참고).
- 관련 코드 전부 `portfolio.js`로 분리됨(기존엔 `index.html`에 흩어져 있었음). 파일 안에 "Google Sheets 연동 시작/끝" 구역이 명확히 구분되어 있어, 나중에 시트 연동을 완전히 버리고 싶으면 그 구역만 삭제하면 됨.
- "시트에서 가져오기"(1회용, 시트 데이터를 웹으로 import) / "구글시트에 백업"(수동 export) 버튼 존재. **백업 버튼은 현재도 실패 중 — 원인 미해결** (셀 한도 정리했는데도 안 됨, 아래 다음할일 참고)
- 시트 값과 웹 값이 다르면 보기화면 빨간 점(●), 편집화면 빨간 입력칸으로 표시만 함(자동 덮어쓰기 없음).

### 알아둘 것 (반복 방지)
- **localStorage는 브라우저/기기별로 분리**됨 — PC에서 입력한 포트폴리오 데이터는 폰/다른 브라우저에 안 보이는 게 정상(버그 아님).
- **코드스페이스에서 작업 후 반드시 커밋**할 것 — 닫히면 워킹트리 변경분이 통째로 사라짐. (단, 사라진 것처럼 보였던 사례 중 실제로는 다 저장돼 있었던 경우도 있었음 — 지레짐작하지 말고 git log/dangling commit까지 확인 후 결론낼 것)

- **연금 타일 PC/모바일 미동기화 문제**: localStorage가 기기별로 분리되는 구조적 한계(포트폴리오와 동일) → 구글시트 백업이 고쳐지기 전까지 쓸 수 있는 임시 해법으로, 관리자 화면에 "내보내기/가져오기" 코드 복사 방식 동기화 추가(클립보드 복사 → 다른 기기에 붙여넣기)

## 에러 이력 (반복 방지용 — 같은 증상 나오면 여기부터 볼 것)

| 에러/증상 | 원인 | 해결 |
|---|---|---|
| 국내 시세 등락률이 실제와 크게 다름(예: SK하이닉스 +26.53%) | 야후 `meta.chartPreviousClose`가 한국종목에서 부정확 | 국내는 네이버금융으로 소스 전환(2026-06-22) |
| 차트 기준선이 종목마다 바닥/천장에 붙음 | min/max 단순스케일이라 기준선이 자연스럽게 가장자리로 밀림 | 최소여백(15%)만 확보, 강제 중앙고정은 하지 않음(빈공간 생기는 부작용) |
| 미디어탭이 갑자기 텅 빔 | 유튜브 RSS 전체채널 차단시 빈 배열로 기존 data.json 덮어씀 | 기존 데이터와 병합하도록 수정 |
| 텔레그램/이메일 중복발송 | cron-job.org 도입 후에도 GitHub native schedule이 같이 남아있어 동시 실행 | `update.yml`의 `on.schedule` 완전 제거 |
| 구글시트 "백업" 실패 | 워크북 셀 1,000만 한도 초과(`_RAW` 탭들) → 정리했는데도 여전히 실패 중, 원인 미해결 | (조사중) |
| 포트폴리오 종목이 기기마다 다르게 보임 | localStorage가 브라우저/기기별 분리 저장(설계상 의도된 동작) | "시트에서 가져오기"로 매번 채우거나, 기기 의식 |
| api_status_404 | 모델명 오류 | claude-haiku-4-5-20251001 사용 |
| WorksheetNotFound | 탭 이름 불일치 | "주식현황상세" 정확히 일치해야 함 |
| 유튜브 할루시네이션 | 자막/설명 부족시 AI가 내용 생성 | 자막 200자 미만이면 제목 기반 요약으로 전환 |
| rainbow 종목 가격 오류 | 277810.KS(코스피)로 잘못 설정 | 277810.KQ(코스닥)로 수정 |
| 매일경제 RSS 파싱 오류 | Cloudflare가 기본 UA 차단 | 풀브라우저 User-Agent로 교체 |
| 12시에만나요 영상 오매칭 | RSS 최신 1건만 사용, 다른 시간대 프로그램 혼입 | RSS 전체 entry 순회 + titleKeyword 필터 |
| Supadata 429 limit-exceeded | 구버전 Apps Script가 매시간 호출해 Free플랜 소진 | 중복호출 방지 추가, 유료플랜은 보류중 |
| data.json push 충돌 | 여러 실행 동시 종료시 git push 경쟁 | pull --rebase 재시도 루프(최대5회) |
